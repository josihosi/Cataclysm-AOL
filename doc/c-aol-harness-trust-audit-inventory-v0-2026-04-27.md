## Verdict

This document is the first state-boundary for the harness trust audit: it inventories the current harness surface and names what each surface claims, what artifact can prove it, and where false confidence can still hide.

It does **not** make current harness scenarios green. Until the later step-ledger/proof-freeze phase lands, current scenario reports should be read as at most yellow unless their own report contains step-local screenshots/metadata proving the exact feature path.

## Provisional same-save policy for the audit phase

Use `dev-harness` / `McWilliams` with the copied disposable fixture/profile `mcwilliams_live_debug_2026-04-07` as the canonical audit anchor where a primitive can be tested there. It is the common denominator for current Smart Zone, Basecamp, ambient/chat, and several bandit scenarios, and it is harness-owned copied state rather than Josef's personal playtest state.

Allowed exceptions must be named in the proof matrix before use:

- transformed bandit fixtures (`bandit_live_world_*`, `bandit_local_*`, `bandit_extortion_*`) when the primitive genuinely needs overmap-special placement, sortie clocks, fire/smoke fields, or controlled contact state;
- old `dev` locker/patrol fixtures when auditing those legacy primitive contracts specifically;
- temporary `tmp.*` bandit scenarios only as workbench evidence, not closure proof, unless promoted and given provenance.

Every later proof run should record: source commit, scenario/profile, fixture/profile snapshot, world, character/save id when available, run directory, applied save transforms, and whether the run used the canonical anchor or an exception.
