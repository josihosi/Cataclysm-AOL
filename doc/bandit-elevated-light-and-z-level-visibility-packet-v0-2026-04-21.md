# Bandit elevated-light and z-level visibility packet v0 (2026-04-21)

Status: checkpointed / done for now.

## Greenlight verdict

Josef explicitly wanted the broader post-benchmark playtesting-prep set greenlit, not only the minimum polite pair in chat.
This packet was the visibility-side follow-through behind the active handoff interaction packet, and it now closes honestly on the current tree.

The point here is to clear the most obvious remaining world-perception weirdness before broader human playtesting starts producing bottomless debug-note soup.
The current product lean is already known:
- nearby cross-z visibility should behave sensibly
- elevated exposed light may deserve longer effective visibility and attraction reach than ordinary ground-hidden light
- a strong exposed elevated fire in a dead dark world, for example a fire on a radio tower, should be visible from very far away instead of being treated like an upstairs candle
- the same meaningful light should matter to zombie-horde pressure in the deterministic proof packet, not only bandits; this does **not** mean the live horde system already sees fires by light

Together with the active handoff packet ahead of it, this packet forms the current playtesting-readiness train.

## Scope

1. Land one bounded elevated-light and z-level visibility packet on the current bandit playback / proof seam.
2. Preserve the already-frozen directional-light footing and extend it only where needed to cover elevated exposure and nearby cross-z reality honestly.
3. Prove nearby cross-z visibility sanity explicitly, for example cases like a visible basement fire from ground level or visible exposed light on `z +1` / `z +2` from `z 0` when line and exposure support it.
4. Prove that elevated exposed light can reach farther or stay legible longer than ordinary hidden light under the same broader footing, without turning every vertical difference into magical global sight.
5. Include at least one flagship long-range case for a strong exposed elevated fire in a dead dark world, for example a radio-tower fire, with an honest expectation that it is visible from very far away and can create shared responder pressure instead of being treated like a tiny local curiosity.
6. Prove that the same meaningful elevated light can carry abstract zombie-horde pressure on the deterministic proof seam too instead of becoming private bandit-only theater; do not claim live horde attraction without a live bridge.
7. Keep smoke honest on the vertical axis too: do **not** smuggle in a rule where smoke automatically gets magical extra general visibility just because the source changed floors.
8. Carry explicit per-scenario goals, tuning metrics, and reviewer-readable outputs so the packet can be judged as product behavior rather than debugger folklore.
9. Extend deterministic coverage so the named scenarios and rendered packet output are proven directly on the current tree.

## Non-goals

- a broad world visibility rewrite
- a full local/overmap handoff redesign, which belongs to the active packet ahead of this one
- fresh concealment architecture beyond what is needed to state the visibility cases honestly
- full zombie tactical simulation
- magical universal z-level sight or a general vertical physics treatise
- reopening already checkpointed directional-light, weather, or smoke packets just because they are nearby

## Success shape

This item is good enough when:
1. one bounded elevated-light and z-level visibility packet exists on the current bandit proof seam
2. deterministic proof shows nearby cross-z visibility behaves sensibly instead of collapsing into floor-bound amnesia
3. deterministic proof shows elevated exposed light can become legible/actionable at longer range under the right conditions without exploding into omniscient global sight
4. a flagship exposed-high-fire scenario, for example a radio-tower fire in a dead dark world, proves genuinely long-range visibility instead of timid toy-local range
5. the matching deterministic scenarios prove the same elevated light can carry abstract zombie-horde pressure too instead of living in a private bandit bubble; live horde attraction remains a separate bridge
6. deterministic proof also shows smoke does **not** gain magical extra general reach merely from floor changes
7. reviewer-readable output exposes the visibility read and benchmark outcomes clearly enough that later playtesting can argue about tuning instead of first principles
8. the slice stays bounded and does not widen into broad architecture, full world-sim theater, or orbital-surveillance nonsense for every tiny upstairs light

## Validation expectations

- use real multi-turn scenario proof on the current playback seam, up to `500` turns where needed
- keep explicit per-scenario goals and tuning metrics, not one global quota hand-waved across unlike cases
- explicitly include one exposed-high-fire benchmark, for example a radio-tower fire in a dead dark world, so the packet has at least one case where "visible from far away" is a required product truth instead of a vague later hope
- deterministic coverage should check the rendered packet or explicit state/report fields directly where possible
- if a scenario misses its benchmark, treat that as routing / perception logic evidence first, not as a cue to soften the benchmark by habit

## Product guardrail

The point is to make the world stop feeling fake at the obvious edges.
If the player sees a strong elevated light, the world should admit that somebody else might notice too.
A fire on a radio tower in a dead dark world should read as a long-range event, not a coy little local secret.
And if a nearby fire or exposed light is one floor off, the system should not behave like a staircase is an amnesia beam.
